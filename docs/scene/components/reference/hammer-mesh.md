# Hammer Mesh

> **Hammer is being deprecated.** For the modern path see **[Scene Mapping](../../scene-mapping/index.md)**. This page documents the legacy Hammer-driven mesh component.

> **Which should I use — [`MeshComponent`](meshcomponent.md) or `HammerMesh`?**
>
> | Use | Component | Authored by |
> |---|---|---|
> | **New project, scene-first mapping** | [`MeshComponent`](meshcomponent.md) | The in-scene [Mapping](../../scene-mapping/index.md) tool with [`PolygonMesh`](polygonmesh.md) data |
> | **Legacy `.vmap` map, brushes tied to entities** | `HammerMesh` (this page) | Hammer's "tie to entity" workflow at `.vmap` compile time |
>
> Hammer is being deprecated in favor of [Scene Mapping](../../scene-mapping/index.md). For new work, prefer `MeshComponent`. `HammerMesh` is the bridge that keeps existing Hammer-authored maps working.

The `HammerMesh` component automatically manages rendering and physics collision for geometry created and tied to GameObjects inside the Hammer map editor.

## Quick Working Example

You typically do not add a `HammerMesh` directly in code, as it is generated by Hammer. However, you can access and modify it at runtime.

```csharp
public class MapGeometryModifier : Component
{
	protected override void OnStart()
	{
		// Find all hammer meshes in the scene
		var hammerMeshes = Scene.GetAllComponents<HammerMesh>();
		
		foreach ( var mesh in hammerMeshes )
		{
			// Change the color tint of the map geometry
			if ( mesh.UseRenderer )
			{
				mesh.Tint = Color.Green;
			}
			
			// Turn the map geometry into a conveyor belt!
			if ( mesh.UseCollision )
			{
				mesh.SurfaceVelocity = new Vector3( 200f, 0, 0 );
			}
		}
	}
}
```

### Setting up Triggers in Hammer

A very common use case for `HammerMesh` is creating invisible trigger volumes.

1. Create a mesh in Hammer and apply the `tools/toolstrigger` material to it.
2. Tie the mesh to a GameObject.
3. In the Inspector for the `HammerMesh` component, disable the **Renderer** feature (right-click the tab or toggle it).
4. In the **Collision** tab, check the **Is Trigger** box.
5. You can now use the `OnTriggerEnter` and `OnTriggerExit` actions to execute logic when a player walks into the volume.

### Conveyor Belts

You can use the `SurfaceVelocity` property to make the surface of the map geometry push physically simulated objects, like a conveyor belt or a rushing river.

```csharp
// Get the component (e.g. on a trigger volume)
var mesh = Components.Get<HammerMesh>();

// Push objects 100 units forward per second along the X axis
mesh.SurfaceVelocity = new Vector3( 100f, 0, 0 );
```

## Configuration

### Renderer

These properties are active when `UseRenderer` is true.

| Property | Description |
|:---|:---|
| `Tint` | The color tint applied to the rendered map geometry. |
| `RenderType` | Determines how the geometry casts shadows (e.g., On, Off, ShadowsOnly). |

### Collision

These properties are active when `UseCollision` is true.

| Property | Description |
|:---|:---|
| `Static` | If true, the collider is considered static and immovable by physics. |
| `Friction` | Overrides the surface friction of the geometry (0 to 1). |
| `Surface` | The physical surface properties (sounds, effects) applied to this geometry. |
| `SurfaceVelocity` | The local velocity of the surface, pushing objects that rest on it (like a conveyor belt). |
| `IsTrigger` | If true, the geometry acts as a non-solid trigger zone. |
| `OnTriggerEnter` | Action called when a collider enters this geometry (if it is a trigger). |
| `OnTriggerExit` | Action called when a collider exits this geometry (if it is a trigger). |

## Troubleshooting

:::info Enabling/Disabling Features
The Renderer and Collision features are optional and can be turned off entirely to save performance. Right-click on the Renderer or Collision tab headers in the Inspector to disable them.
:::

:::warning Trigger Events Not Firing
If you checked `IsTrigger` but your events aren't firing, ensure that the object entering the volume has a `Rigidbody` or `Collider` component that is capable of interacting with triggers, and that collision tags between the two objects allow them to touch.
:::
